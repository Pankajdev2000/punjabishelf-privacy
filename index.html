Get Your Razorpay Live Key
Payment won't work without this
Where to find it

🌐 Go to Razorpay Dashboard

dashboard.razorpay.com/app/keys
SETTINGS → API KEYS Regenerate Keys
KEY ID (this is what you need)

KEY SECRET (keep secret, not used in app)

Go to dashboard.razorpay.com → Settings → API Keys → Generate Live Keys

⚠️ Make sure you switch from Test Mode to Live Mode in the top bar before generating keys. Test keys start with rzp_test_ — live keys start with rzp_live_

2
Generate Release Keystore
Google Play rejects apps signed with debug keys
🚨 Do this ONCE and keep the .jks file safe forever. If you lose it, you can never update your app on Play Store. Back it up to Google Drive or a USB drive.
Step 2a — Generate the keystore

💻 Run this command in your terminal

Open terminal anywhere on your PC and run:

Terminal 
keytool -genkey -v \
  -keystore punjabishelf-release.jks \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -alias punjabishelf

It will ask you questions — fill them in:

Enter keystore passwordChoose any password. Write it down!

What is your first and last name?Your full name
What is your organizational unit?Can leave blank — press Enter
What is the name of your organization?PunjabiShelf
What is the name of your city?Your city (e.g. Amritsar)
What is the name of your state?Punjab
What is the two-letter country code?IN
Enter key password for aliasSame password as keystore (press Enter)
✅ This creates punjabishelf-release.jks in your current folder. Move it into your project's android/ folder.
Step 2b — Fill in key.properties

📁 Open this file: android/key.properties

android/key.properties 
# Fill these in with YOUR actual values:
storePassword=YourChosenPassword
keyPassword=YourChosenPassword
keyAlias=punjabishelf
storeFile=../punjabishelf-release.jks
⚠️ Never push key.properties or the .jks file to GitHub. They are already in .gitignore in your project.
3
Get SHA-1 Fingerprint
Required for Google Sign-In to work
Step 3a — Get debug SHA-1 (for testing)

💻 Run in terminal

Terminal — Debug SHA-1
# Windows:
keytool -list -v \
  -keystore %USERPROFILE%\.android\debug.keystore \
  -alias androiddebugkey \
  -storepass android -keypass android

# Mac/Linux:
keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android -keypass android

Look for the line that says SHA1: — copy that value.

Step 3b — Get release SHA-1 (for production)

💻 Run in terminal (after generating keystore in Step 2)

Terminal — Release SHA-1
keytool -list -v \
  -keystore android/punjabishelf-release.jks \
  -alias punjabishelf

5
Deploy Firebase Security Rules
Protect your database from unauthorized access
Option A — Firebase CLI (recommended)

💻 Deploy from terminal

Terminal — from project root
# Install Firebase CLI (once):
npm install -g firebase-tools

# Login:
firebase login

# Deploy rules only:
firebase deploy --only firestore:rules,storage
Option B — Firebase Console (manual)

🔥 Paste rules directly in browser

Go to Firebase Console → Firestore Database → Rules tab → paste the contents of firestore.rules → click Publish.

Then go to Storage → Rules → paste storage.rules → Publish.

6
Host Your Privacy Policy
Play Store requires a live URL — not just a file
Free hosting via GitHub Pages

📄 5-minute setup

  • 1️⃣Go to github.com → Create new repository named exactly punjabishelf.github.io
  • 2️⃣Create a folder named privacy-policy in the repo
  • 3️⃣Upload the privacy-policy.html file from your project ZIP as index.html inside that folder
  • 4️⃣Go to repo Settings → Pages → Source = main branch → /root → Save
  • 5️⃣Your policy is now live at: https://punjabishelf.github.io/privacy-policy
✅ This URL is already hardcoded in your app (Profile → Privacy Policy). No code change needed.
7
Build Release AAB
AAB is what Play Store accepts — not APK
Run these commands in your project folder

💻 Terminal — in your Flutter project root

Terminal
# 1. Clean old build files:
flutter clean

# 2. Get all packages:
flutter pub get

# 3. Build release AAB:
flutter build appbundle --release
📦 Your AAB file will be at: build/app/outputs/bundle/release/app-release.aab — this is the file you upload to Play Console.

🎨 What to prepare before submitting

  • 📱Screenshots — at least 2 phone screenshots (1080×1920px). Take them from your running app on your phone.
  • 🖼️Feature graphic — 1024×500px banner image for the Play Store listing page.
  • 🔷Hi-res icon — 512×512px PNG. Your app_icon.png from assets can be used if it's 512px.
  • 📝Short description — max 80 characters. Example: "Punjabi digital library — books, poetry & literature"
  • 📄Full description — max 4000 characters. Describe the app, features, premium benefits.
  • 🏷️Category — Books & Reference
